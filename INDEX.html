📚 Endpoints da API

1. Listar Eventos (Paginado)

GET /api/events?page=0&size=10

Parâmetros:

  • page (opcional): Número da página (padrão: 0)
  • size (opcional): Itens por página (padrão: 10)

Exemplo cURL:

curl http://localhost:8080/api/events

2. Obter Evento por ID

GET /api/events/{id}

Exemplo:

curl http://localhost:8080/api/events/1

3. Criar Evento

POST /api/events

Body (JSON):

{
  "title": "Workshop Angular",
  "description": "Workshop sobre Angular 17",
  "eventDateTime": "2026-05-20T14:00:00",
  "location": "São Paulo"
}

Exemplo cURL:

curl -X POST http://localhost:8080/api/events \
  -H "Content-Type: application/json" \
  -d '{"title":"Novo Evento","description":"Desc","eventDateTime":"2026-05-20T14:00:00","location":"Local"}'

4. Atualizar Evento

PUT /api/events/{id}

Body: Mesmo formato do POST

5. Deletar Evento

DELETE /api/events/{id}

Retorna: 204 No Content

🔧 Como Iniciar

1. Compilar e rodar testes:

mvnw.cmd clean test

2. Iniciar a aplicação:

mvnw.cmd spring-boot:run

Ou usando o JAR compilado:

java -jar target/eventos-0.0.1-SNAPSHOT.jar

3. Acessar a aplicação:

http://localhost:8080/api/events

ℹ️ Informações Importantes

  • A API usa banco de dados H2 em memória por padrão (perfeito para testes)
  • Para usar PostgreSQL em produção, configure as variáveis de ambiente
  • Todos os endpoints requerem Content-Type: application/json
  • As datas devem estar em formato ISO 8601: YYYY-MM-DDTHH:mm:ss
  • Soft delete implementado (eventos marcados como deletados, não removidos)
